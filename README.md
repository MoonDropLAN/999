## 用户旅程地图

| 阶段 | 用户目标 | 关键触点 | 用户动作 | 系统动作 | 情绪 / 风险 | 设计机会 |
|---|---|---|---|---|---|---|
| 1. 初次配置 | 把正在吃的药放进系统 | 手机拍照、电子药箱 | 拍药盒正面，确认药名、规格、频次 | OCR 识别包装，匹配药品库，生成候选计划 | 担心识别错、不会设置 | 大字号确认页；剂量和频次必须二次确认；错误字段可语音修改 |
| 2. 日常提醒 | 按时服药 | 手机通知、药箱灯光 / 声音 | 点击已服用、稍后提醒或漏服 | 定时提醒，记录状态，同步家属端 | 容易忘、被打扰 | “稍后提醒”比“关闭”更安全；提醒语短、明确、可朗读 |
| 3. 用药理解 | 知道药为什么吃、有什么注意事项 | AI 问答 | 提问“漏服怎么办”“能不能一起吃” | 基于说明书和风险规则回答，必要时建议咨询医生 | 对副作用焦虑 | 回答先给行动建议，再给原因；高风险症状明确升级 |
| 4. 家属协同 | 家属知道是否按时服药 | 家属端摘要 | 查看今日服药状态 | 异常漏服、库存不足时推送 | 害怕打扰老人 | 只推送异常，不实时监控；尊重老人自主性 |
| 5. 复诊 / 买药 | 带着准确记录看医生 | 服药记录、库存提醒 | 导出记录或补充购药 | 汇总依从性、副作用、漏服次数 | 记录分散 | 一键生成复诊摘要；库存低于阈值提前提醒 |

## 核心 Demo 范围

1. **拍照上传药盒包装**
   - 通过模拟取景框识别“缬沙坦胶囊 80mg”。
   - 识别结果展示药名、规格、建议计划、库存和风险提醒。
   - 用户点击“确认放入药箱”后生成提醒任务。

2. **定时提醒与服用状态记录**
   - 今日服药卡片展示时间、药名、剂量、服药说明。
   - 支持“确认已服”和“稍后提醒”。
   - 更新今日完成数，并模拟同步家属端。

3. **用药知识问答和副作用提醒**
   - 内置漏服、副作用、联合用药三个高频问题。
   - 回答遵循安全边界：不替代诊断、不直接建议停药换药、高风险症状升级。

## 关键配置示例

```yaml
system_prompt:
  role: 面向 55-75 岁用户的 AI 用药伴侣
  goals:
    - 准确记录药品与服药计划
    - 提醒按时服药并记录状态
    - 提供说明书级别的用药知识解释
  guardrails:
    - 不替代医生诊断
    - 不直接调整处方剂量
    - 识别结果必须让用户确认
    - 严重症状触发就医提醒
```

```yaml
workflow:
  scan_package:
    input: camera_image
    output: confirmed_medication_plan
  reminder:

  qa:
    sources: [drug_label, interaction_rules, user_medication_profile]
    escalation: [severe_allergy, breathing_difficulty, abnormal_blood_pressure, repeated_missed_dose]
```

## 最重要验证假设

**最重要的假设：55-75 岁中老年用户能否在没有子女帮助的情况下，准确完成“拍药盒 -> 确认药品信息 -> 生成提醒计划”。**

原因：如果初次入库失败，后续提醒、药箱记录、家属协同和问答都会建立在错误数据上。这个假设同时验证 OCR 识别准确度、确认页可理解性、老人操作负担和安全风险。

建议验证方式：

1. 招募 8-12 位 55-75 岁、多药共服用户，用真实药盒完成入库任务。
2. 观察是否能在 3 分钟内独立完成，并记录在哪些字段停顿或误点。
3. 核心指标：药品识别正确率、剂量/频次确认正确率、任务完成率、平均完成时间、用户信心评分。
4. 通过家属或药师复核入库结果，确认是否存在会导致错误服药的严重错误。
